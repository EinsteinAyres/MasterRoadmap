
- A regra do escoteiro
- Mantenha classes, métodos e arquivos pequenos
- Comente apenas o necessário
- Use nomes significativos
- Formatação e estilo de código
- Evite abstrações precipitadas e refatore sempre
- Minimize a complexidade
- Prefira exceções a retorno de códigos de erro
- Seja consistente
- O que são Testes de Unidade
- Evite passar booleans e nulls
- O que são Funções puras
